Battle-Test ·

Commissioning field tool — automated pre-release chaos testing at field scale

Run the real shipped app against a real field dataset, hammer it with injected faults for hours, and let machine-checked invariants decide pass or fail — before a tablet ever sees the build.

✅
Nightly soak armed — runs 02:00 → 10:00 every night on the CI runner 8-hour all-chaos run · red pipeline + 30-day artifacts if any invariant fails

Architecture

One docker compose project. No production data is touched — the cloud is a throwaway Postgres seeded from a copy of field data and deleted after each run.

Bottom line: the failure classes that have actually bitten the field — scale freezes, leaks, PLC-download data loss, and silent sync loss — are now reproduced automatically every night with a no-loss verdict. The amber rows are simulated well enough to catch logic bugs, not hardware-specific ones. Roadmap to raise them: a Playwright crew (real UI), packet-level network shaping, and a periodic run against the real Logix Emulate box.
