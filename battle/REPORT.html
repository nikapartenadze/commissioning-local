1 · What & why

Field reports: “syncs disappear / not pushed” and “the tool says synced but the cloud is missing data.” Root investigation tied this to the TPA8/MCM11 incident class — results testers entered were silently lost, while the UI looked clean. Two structural gaps made it invisible:

  • The cloud rejected/throttled writes and the tool deleted the queued result instead of keeping it.
  • The “unsynced” badge only counted queue rows, so a deleted row read as “synced.”

We also stood up an automated battle-test environment so this class of bug is reproduced and caught before it reaches a tablet — at field scale, for hours, under injected chaos, with machine-checked invariants.

4 · How the env caught new issues (the rigor)

① A release‑blocker unit tests could not see

While verifying v2.40.4, the env failed on a fresh database boot: a new DB trigger referenced the DeadLettered column before the migration that adds it, so the whole schema init threw and silently skipped every migration — the entire fix was dead and sync errored each cycle. 280 passing unit tests did not catch this — only a real fresh‑tablet startup did. Fixed before ship.

② A “FAIL” verdict run to ground instead of hand‑waved

A run reported 298 “data losses.” Rather than accept it, we converged the system (stopped the bots, drained the queue to 0) and compared local vs cloud directly:

Zero real loss. Two of our own test invariants were wrong (I1 counted a one‑time boot transient; I4 treated last‑write‑wins as loss) — both fixed so the verdict is trustworthy.

7 · Net outcome

  • Field data is no longer silently lost. Rejected/stuck results are parked (kept + reason), shown as a red CHECK marker, written to a 14‑day audit log, and reported in the heartbeat — visible, durable, recoverable, instead of invisible for days.
  • The badge can’t lie — it reconciles real state, not just a queue count; a failed status read shows “unknown,” never “synced.”
  • The cloud no longer throttles legitimate field pushes at a busy multi‑subsystem site.
  • Faster sync — the queue can’t bloat (per‑IO coalescing).
  • A standing safety net — every release now runs through a field‑scale chaos soak with machine‑checked no‑loss invariants, which already caught two bugs that would otherwise have shipped.
